# project.sh - Flutter 项目管理脚本使用说明

## 1. 脚本概述

`project.sh` 是一个用于 Flutter 项目的管理脚本，用于更新 Flutter 项目的依赖和配置，包括升级 pub 依赖和 Gradle 版本。

### 版本信息
- **版本**: 1.0.0
- **最后更新**: 2026-03-23
- **作者**: Stone

## 2. 功能特点

- **依赖管理**: 升级 pub 依赖到最新版本
- **Gradle 升级**: 自动升级 Gradle 版本到推荐版本
- **环境检查**: 检查 Flutter 安装状态
- **详细日志**: 提供详细的执行过程日志和错误处理

## 3. 安装和设置

### 前提条件
- Flutter 环境已安装
- 项目中已配置 `pubspec.yaml` 文件

### 安装步骤
1. 将 `project.sh` 脚本复制到项目的 `lib/sh` 目录中
2. 赋予脚本执行权限：
   ```bash
   chmod +x lib/sh/project.sh
   ```

## 4. 目录结构

脚本会操作以下文件：

```
项目根目录/
├── pubspec.yaml          # Flutter 项目依赖配置文件
└── android/gradle/wrapper/gradle-wrapper.properties  # Gradle 配置文件
```

## 5. 命令行参数

| 参数 | 描述 |
|------|------|
| `--update` | 更新 Flutter 项目依赖和配置 |
| `--help` | 显示帮助信息 |
| `--version` | 显示脚本版本信息 |

## 6. 使用示例

### 6.1 更新项目依赖和配置

**功能**: 升级 pub 依赖和 Gradle 版本

**命令**: 
```bash
./project.sh --update
```

**执行流程**:
1. 检查 Flutter 是否安装
2. 获取当前 Flutter 版本
3. 升级 pub 依赖
4. 升级 Gradle 版本
5. 显示更新结果

## 7. 常见问题和解决方案

| 问题 | 解决方案 |
|------|----------|
| Flutter 未安装 | 安装 Flutter 并添加到 PATH |
| pubspec.yaml 文件不存在 | 确保在 Flutter 项目目录中执行脚本 |
| Gradle 升级失败 | 检查 Android 目录是否存在，以及 gradle-wrapper.properties 文件是否可写 |
| 权限不足 | 确保当前用户有写入文件的权限 |

## 9. 总结

`project.sh` 脚本是一个实用的 Flutter 项目管理工具，通过自动升级依赖和配置，确保项目使用最新的依赖版本和构建工具。使用此脚本可以：

- 确保项目依赖保持最新状态
- 自动升级 Gradle 版本，适应 Flutter 的最新要求
- 减少手动管理依赖和配置的工作量
- 提高项目的稳定性和兼容性
